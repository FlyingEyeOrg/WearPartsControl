# UI 标准设计规范

本文档用于约束 `WearPartsControl` 项目的 WPF 界面设计实现，减少页面级样式漂移，保证按钮、表单、对话框以及多语言场景下的视觉一致性。

## 1. 目标

- 统一同类控件的视觉尺寸与交互节奏。
- 优先复用项目级共享样式，避免页面内零散硬编码。
- 保证中文、英文等多语言切换后不出现文本裁剪、按钮忽宽忽窄、布局塌陷。
- 保持界面风格简洁、克制、接近系统级业务工具，而不是堆叠临时样式。

## 2. 基本原则

### 2.1 共享样式优先

- 新增 UI 时，优先使用 `Resources/ShellContentStyles.xaml` 中的共享样式。
- 优先使用 HandyControl 已有按钮样式：`ButtonPrimary`、`ButtonDefault`、`ButtonDanger`。
- 页面中如果出现重复的按钮尺寸、间距、排版需求，应先抽到共享样式，再落地到页面。
- 禁止为了单个页面视觉微调，大量引入局部 `Width`、`MinWidth`、`Padding` 的硬编码副本。

### 2.2 视觉一致性优先于局部“看起来差不多”

- 相同语义、相同层级的按钮，宽度必须一致。
- 不允许同一页面或不同页面中，两个字按钮宽度分别为 `92`、`96`、`120`、`128`、`156`、`160` 这类历史残留值。
- 若按钮属于同一交互层级，应使用同一套共享规则，而不是由页面作者自行决定宽度。

### 2.3 多语言安全优先

- 不能靠压缩按钮宽度、固定标签宽度、裁剪文本来“适配英文”。
- 应优先让容器可换行、可拉伸、可自适应，而不是让文案被截断。

## 3. 按钮规范

## 3.1 标准操作按钮宽度

- 项目内标准操作按钮的最小宽度统一为 `120`。
- 适用范围：登录、保存、取消、刷新、查询、关闭、导入、测试、恢复、退出等常规操作按钮。
- 长文案按钮允许在 `MinWidth="120"` 基础上按内容自然增宽，不额外限制固定宽度。

### 3.2 宽度使用规则

- 优先使用 `MinWidth="120"`，不使用固定 `Width` 限死按钮宽度。
- 不再新增以下历史宽度写法：`92`、`96`、`128`、`156`、`160`。
- 若业务确实需要比标准按钮更宽，必须先确认是否属于独立组件规范，再抽取共享样式后使用。
- 不允许因为某个按钮文字较短，就把它单独缩窄成“更紧凑”的局部样式。

### 3.3 样式复用规则

- 工具栏主操作按钮优先使用共享样式 `ShellToolbarPrimaryButtonStyle`。
- `ShellToolbarPrimaryButtonStyle` 当前标准最小宽度为 `120`，不得在具体页面再次覆盖成更小值。
- 普通确认按钮使用 HandyControl 基础样式配合统一 `MinWidth="120"`。
- 危险操作按钮使用 `ButtonDanger`，但按钮宽度仍遵守标准宽度规则。

### 3.4 Padding 规则

- 不再通过统一增加 `Padding="14 6"` 之类的方式制造“按钮更大”的视觉效果。
- 若按钮文本已存在裁剪风险，优先调整高度、垂直对齐和最小宽度，而不是继续堆叠水平内边距。
- 页面内若出现按钮文字被截断，先检查按钮容器和按钮高度，不要先加大 Padding。

## 4. 表单与操作区规范

### 4.1 表单行布局

- 表单标签列优先使用 `Grid.IsSharedSizeScope="True"` 和 `SharedSizeGroup` 统一对齐。
- 标签文本必须允许换行：`TextWrapping="Wrap"`。
- 不使用固定 `80`、`120`、`160` 等标签宽度硬挤多语言文案。

### 4.2 操作区布局

- 窗口底部操作区应固定为独立操作容器，视觉上与内容区分离。
- 操作按钮统一右对齐或按组件设计规范居中，不允许同一类弹窗出现混乱对齐。
- 同一操作区中的主次按钮，宽度应保持一致；主按钮通过颜色/样式区分，不通过额外拉宽区分。

### 4.3 工具栏布局

- 查询区、筛选区、列表工具栏优先使用 `WrapPanel`，避免在英文环境下横向挤爆。
- 工具栏按钮必须使用共享样式，避免单页内重复定义边距和宽度。

## 5. 对话框规范

### 5.1 对话框结构

- 推荐结构：标题区、内容区、底部操作区。
- 底部操作区必须始终位于内容底部，不因内容多少而漂移到中间。
- 对话框组件避免额外装饰条或与业务无关的视觉元素，保持简洁。

### 5.2 对话框按钮

- 对话框中的确认、取消、关闭等按钮，统一使用标准操作按钮宽度。
- 禁止“保存 128、取消 96、关闭 120”这类同层级按钮宽度不一致的写法。

## 6. 登录与系统级入口界面规范

- 登录窗口应保持轻量、克制，不使用与业务无关的重装饰背景。
- 登录按钮、登出按钮、需要登录提示区按钮，均遵守标准按钮宽度规则。
- 登录类入口控件应优先保证信息层次清晰、状态文本可读、按钮尺寸一致。

## 7. 多语言与文本显示规范

### 7.1 禁止事项

- 禁止依赖 `TextTrimming` 作为主要的多语言适配手段。
- 禁止通过缩窄按钮、缩窄标签列、缩窄弹窗来强行容纳英文文案。
- 禁止为了“当前中文看起来更紧凑”，牺牲英文布局可读性。

### 7.2 推荐做法

- 文本优先允许换行展示。
- 容器优先使用 `MinWidth`、`SizeToContent`、`WrapPanel`、共享列宽等方式提升弹性。
- 按钮宽度统一后，长文本按钮自然扩展，不额外单页调宽。

## 8. 落地要求

### 8.1 新增或修改页面时

- 新页面中的常规操作按钮默认按 `MinWidth="120"` 设计。
- 工具栏主按钮默认使用 `ShellToolbarPrimaryButtonStyle`。
- 页面评审时需要检查是否出现新的局部按钮宽度常量。

### 8.2 代码审查检查项

- 是否复用了共享样式，而不是页面内重复定义按钮尺寸。
- 是否引入了新的固定按钮宽度常量。
- 同类按钮在同页及跨页是否保持一致。
- 英文或长文本场景下是否仍可正常显示，无裁剪、无重叠、无布局错位。

## 9. 当前项目已采用的按钮宽度结论

- 标准最小宽度：`120`
- 共享工具栏主按钮样式：`ShellToolbarPrimaryButtonStyle`
- 允许扩展方式：在共享样式基础上按内容自然增宽
- 禁止新增的历史残留宽度：`92`、`96`、`128`、`156`、`160`

后续如果确实需要引入新的按钮尺寸层级，必须先更新本规范，并同步抽取为共享样式，不能直接在页面中落地新的硬编码宽度。
